Tekon P0 Provider Smoke

Codex Provider Smoke 手册

用于验证 Tekon --agent codex 能通过本机 Codex CLI 跑通真实 provider run，并最终创建真实 PR 证据。

profile internal artifact manifest human-approved PR only

1. 目标

真实 provider

本机 Codex CLI 可用，并由 Tekon 通过 codex --profile internal ... exec 调起。

结构化产物

Codex 按 TEKON_ARTIFACT_MANIFEST 写回必需 artifact。

可恢复快照

run provider snapshot 记录为 codex，resume 不降级。

真实 PR

只在人工确认后执行 delivery create-pr --approve-human。

2. 官方资料和判断依据

资料 资料内容 对 Tekon 的判断依据
Codex noninteractive codex exec 面向脚本、CI 和一次性自动化任务。 Tekon 可以把 workflow node 映射为一次非交互 provider 调用。
Approvals & security Codex 支持 sandbox、approval 和网络访问边界组合。 Tekon 默认固定 codex --profile internal --sandbox workspace-write --ask-for-approval on-request --add-dir <TEKON_OUTPUT_DIR> exec。 --add-dir 只由 Tekon 受控追加到 artifact 输出目录。
CLI reference CLI 参考列出 exec、sandbox、approval、配置覆盖和危险 bypass 参数。 Adapter 必须拒绝用户覆盖 profile、sandbox、approval、文件系统、配置或危险 bypass。

3. 前置条件

codex --version
codex --profile internal --sandbox workspace-write --ask-for-approval on-request exec --help
npm exec --yes -- pnpm@10.12.1 install --frozen-lockfile
npm exec --yes -- pnpm@10.12.1 build
如果 codex --version 或 internal profile smoke 不可用，先安装并认证 Codex CLI internal profile。不要用 mock 结果替代 Codex provider smoke。

上面的 exec --help 只验证 CLI 与 internal profile 是否可用；真实 Tekon run 会由 adapter 在 exec 前受控追加 --add-dir <TEKON_OUTPUT_DIR>，只开放本节点 artifact 输出目录。

4. 最小运行命令

node packages/cli/dist/index.js init --repo /Users/zhaoensheng/Projects/tekon

node packages/cli/dist/index.js run "补齐 Codex provider smoke 证据，要求产出可审阅文档和测试记录。" \
  --template docs-update \
  --agent codex \
  --repo /Users/zhaoensheng/Projects/tekon

期望输出包含 runId=run_... 与 status=passed。 

node packages/cli/dist/index.js review --repo /Users/zhaoensheng/Projects/tekon
node packages/cli/dist/index.js status --repo /Users/zhaoensheng/Projects/tekon

5. 验证和样本记录

node packages/cli/dist/index.js eval readiness --repo /Users/zhaoensheng/Projects/tekon

node packages/cli/dist/index.js eval work-usability record \
  --id tekon-codex-self-bootstrap \
  --expected-provider codex \
  --require-real-provider \
  --require-pr \
  --samples docs/reviews/tekon-codex-work-usability-samples.yaml \
  --repo /Users/zhaoensheng/Projects/tekon

6. 创建 PR

node packages/cli/dist/index.js delivery prepare --repo /Users/zhaoensheng/Projects/tekon

node packages/cli/dist/index.js delivery create-pr \
  --approve-human \
  --repo /Users/zhaoensheng/Projects/tekon

node packages/cli/dist/index.js delivery ci-status --repo /Users/zhaoensheng/Projects/tekon
ci-status 是只读查询，不 rerun CI、不 merge、不上线。

7. 结果归档

  • 需求摘要、run id 和 provider snapshot。
  • readiness、gate、PR package、PR URL 和 CI status。
  • 人工介入点、风险和后续动作。
  • Markdown 与 HTML 报告归档到 docs/reviews/。

8. 常见失败

失败现象 可能原因 处理
Unsupported agent: codex CLI/Web 未接线 Codex provider。 更新到包含 Codex adapter 的版本并重新构建。
agent failed: provider=codex exitCode=1 manifest 缺失、artifact schema 不合法、命令失败，或用户 args 试图覆盖 profile、sandbox、approval、文件系统、配置或危险 bypass。 查 stdout/stderr、manifest 和 artifact 内容。
resume 拒绝 缺 provider snapshot 或 snapshot 不能 replay。 重新跑真实 Codex run，不要手工改 provider。
PR 等待审批 未传 --approve-human。 审阅 PR 包和 diff 后再显式批准。
