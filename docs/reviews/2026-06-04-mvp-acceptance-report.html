Donkey MVP 验收报告

日期：2026-06-04

验收对象：Donkey Phase 0/1 本地可运行 MVP

结论：验收通过。MVP 已实现 CLI、文件态状态、Local Runner、Intent / Target Stage Gate、Workflow 裁剪、Agent Registry、Tool Gateway、Evidence HTML 报告和 donkey-eval；本地验证通过，reviewer 复查无 Must Fix。

实现范围

能力验收结果说明
CLI通过支持 init、run、status、show、eval。
文件态状态通过每次运行写入 .donkey/runs/<runId>/state.json，包含 schema version、Intent、Workflow、ToolRun、Evidence 和事件摘要。
Intent / Gate通过支持想法、需求、技术方案、任务/变更、PR 和高危需求降级；高危关键词与 Repo Profile 高危路径均参与判断。
Workflow 裁剪通过已有技术方案到测试验收会跳过需求与开发执行阶段，只声明已执行验证；高危需求停止在风险报告。
Agent Registry通过内置 intent-agent、pm-agent、tech-agent、rd-agent、test-agent、review-agent、evidence-agent。
Tool Gateway工程验证通过统一执行命令，默认拒绝未 allowlist 的命令，先拦截 shell 控制符和重定向，再以 shell: false 执行 argv；记录 stdout/stderr、耗时、退出码，并对命令、输入、日志和报告做敏感信息脱敏。
Evidence / HTML通过运行报告与 Eval 报告均生成 HTML 人审版。
donkey-eval工程验证通过内置 5 个 EvalCase replay，通过率 100%，输入类型准确率 100%，目标阶段准确率 100%，高危误放行 0。当前为 Phase 0 内置样本，不等同于 Phase 1 的 5 个真实 B/D 需求验收。

边界说明

  • 本版本是 Phase 0/1 Local Runner MVP，不包含正式 Server、Console、PostgreSQL、Temporal、CI Runner、Container Runner 或真实外部 Coding Agent 调用。
  • 代码变更执行节点保留 Adapter 边界，当前主要完成测试验收、文档产物、风险降级和证据闭环。
  • 内置 Intent/Gate 仍是规则化实现，后续需要用真实需求样本继续扩展 EvalCase。
  • 当前报告记录的是本地 MVP 骨架验收；Phase 1 真实试点仍需补充至少 5 个真实 B/D 类需求样本。
