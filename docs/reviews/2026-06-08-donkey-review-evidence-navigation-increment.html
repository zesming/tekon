# Donkey 审阅证据导航增量报告

日期：2026-06-08

范围：补齐 P0-5 “人工验收面可决策”的第一版深度上下文导航。上一轮 review surface 已经能展示 readiness、diff、artifact 正文、gate log、PR 包和下一步命令，但用户仍需要自己判断某个失败项应该看哪份证据。本轮新增 Evidence Navigation，把 readiness 失败项直接关联到 artifact、gate log、audit event、PR body、PR package 和 delivery diff。

## 1. 背景判断

真实工作场景里，审阅者最需要回答的是“为什么不能交付”和“应该先看哪里”。只有正文和日志堆叠在页面上，仍然会让用户在 artifact、gate、audit 和 PR 包之间人工搜索。本轮不扩大自动化范围，只增强人工判断路径：Donkey 给出证据下一跳，人仍决定是否接受、重试、创建 PR 或返工。

## 2. 已完成能力

| 能力 | 实现位置 | 说明 |
| --- | --- | --- |
| Evidence groups | `packages/core/src/review/surface.ts` | `createWorkReviewSurface` 新增 `evidenceGroups`，为 review route 和 failed readiness checks 生成证据链接。 |
| 证据链接类型 | `packages/core/src/review/surface.ts` | 支持 `artifact`、`gate-log`、`audit-event`、`pr-body`、`pr-package` 和 `diff`，并对链接去重、截断。 |
| CLI 展示 | `packages/cli/src/index.ts` | `donkey review` 新增 `## Evidence Navigation`，把失败项和对应证据位置输出到终端。 |
| Web 展示 | `packages/web/src/client/App.tsx` | dashboard 新增 `Evidence Links` 面板；diff、artifact、gate log、audit row 和 PR 包拥有可跳转锚点。 |
| 回归测试 | `packages/core/__tests__/review/surface.test.ts`、`packages/cli/__tests__/run-cli.test.ts`、`packages/web/__tests__` | 覆盖 core evidence groups、CLI Evidence Navigation、Web API 返回和 Web e2e 页面展示。 |

## 3. 当前边界

- Evidence Navigation 是审阅辅助，不会自动修复、自动重试、自动创建 PR 或自动 merge。
- 关联规则是第一版启发式：按 readiness check 类型映射到相关 artifact、gate、audit 和 delivery 证据。真实 run 后仍需观察是否有误导或缺漏。
- 只读取目标仓库边界内的 artifact/gate/delivery 文件；仍沿用 review surface 的 safe path 约束。
