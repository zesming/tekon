6. 角色评审与职责边界

  • PM 先产出需求卡/PRD，再进行 PM 内部需求评审与需求意图复核。
  • RD/QA 分别进行需求接口评审，只看自身职责范围。
  • RD 在开发前产出 implementation plan，并由独立 RD technical review 评估技术方案。
  • QA 产出 test plan，并通过 QA/PM test plan review 校验测试方案合理性。
  • Reviewer 独立审阅 RD code change。
  • QA validation 与 QA release signoff 做最终“所测即所得”签署。
  • PMO checkpoint 校验节点产出、gate、artifact、pending human decision 与交付边界。
