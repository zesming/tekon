Donkey V2 发布就绪加固实施计划

生成日期：2026-06-05；目标分支：rebuild-v2

目标

在创建 PR 之前完成阶段一剩余发布就绪项：远端 CI 绿灯确认、GitHub Actions lint、Vitest 配置迁移、基础文档、真实 Claude provider 手动 smoke、网络策略能力边界和最终验收报告。

硬门禁

  • 最终提交 push 后，必须确认远端 Core workflow 对该 commit 成功。
  • 真实 Claude provider smoke 必须在人工凭证环境执行成功，并写入脱敏证据。
  • 同时缺少 GITHUB_TOKEN 和已认证 gh，或缺少 Claude CLI / Claude auth 时只能标记 blocked，不能标记完成。
  • PR 创建放在最后，本计划不创建 PR。

Subagent 执行模型

  • 按任务顺序执行，不并行修改共享文件。
  • 每个任务由 worker subagent 执行。
  • 每个任务完成后，启动最高思考等级 reviewer 做 spec compliance 和 code quality 复查。
  • reviewer 检出必须修复项时，先修复再复查。

关键步骤

任务 1

  • 新增 scripts/ci/actionlint.sh，优先使用本地 actionlint，否则用 Docker 镜像 rhysd/actionlint:1.7.12。
  • 新增 scripts/ci/check-github-actions.mjs，无 GITHUB_TOKEN 时 exit 2。
  • workflow 增加 permissions: contents: read、concurrency、actionlint job、Node 24、严格 ignored-builds gate。 
bash -n scripts/ci/actionlint.sh
node --check scripts/ci/check-github-actions.mjs
bash scripts/ci/actionlint.sh
npm exec --yes -- pnpm@10.12.1 --filter @donkey/core build
npm exec --yes -- pnpm@10.12.1 --filter @donkey/core test:unit
npm exec --yes -- pnpm@10.12.1 --filter @donkey/core test:e2e

任务 2

  • 用根 vitest.config.ts 的 test.projects 替代旧 workspace 配置。
  • 创建中文 README、CHANGELOG、docs/manual/donkey-mvp-user-manual.md/html。
  • 手册只说明当前边界，不写不存在的 CLI/Web quickstart。
npm exec --yes -- pnpm@10.12.1 test -- --run
npm exec --yes -- pnpm@10.12.1 build
npm exec --yes -- pnpm@10.12.1 typecheck

任务 3

  • CommandGateway 增加 envMode: safe-default | inherit | exact，默认只传安全环境变量。
  • Claude smoke fail-closed：未设置 DONKEY_CLAUDE_PROVIDER_SMOKE=1 或未通过 claude auth status 时失败。
  • smoke 使用完整 WorktreeLease 和 RunContext 字段。 
DONKEY_CLAUDE_PROVIDER_SMOKE=1 DONKEY_CLAUDE_COMMAND=claude npm run smoke:claude-provider

任务 4

  • network: disabled/restricted 静态拒绝明显网络命令。
  • provider network evidence 使用 declared，不写 os-enforced，除非有 OS/container probe。
  • Claude args 拒绝 --permission-mode、--permission-mode=bypassPermissions 和 --dangerously-skip-permissions。

任务 5

  • 先跑全量本地 gate，再 push 代码完成提交。
  • 必须确认代码完成提交的远端 Core workflow 为 completed/success。
  • 最终报告区分代码完成提交 SHA 和报告提交 SHA；报告提交后的最终远端 workflow URL 写入最终交付说明，不回写报告以避免提交循环。
  • changelog 不得在 reviewer 执行前声明“最高思考等级 subagent review 已通过”。
